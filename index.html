产品范式

「无代码」不是终点，LLM-native 才是真正的产品分水岭

Karpathy 在 Sequoia Ascent 上提出一个关键命题：LLM 的价值远不止「加速已有流程」。他举了三个例子——menugen（全由 LLM 原生处理的图→图应用）、install.md（用自然语言脚本替代 bash）、以及基于非结构化知识的检索系统——来说明一个事实：有些东西 过去根本不可能存在，不是因为技术不够成熟，而是因为古典代码范式无法描述它。当产品第一次能做到「过去不可能的事」，而不只是「更快的事」，才真正迈入了新范式。

查看原文
menugen: an app that can be fully engulfed by LLMs, with no classical code needed

install .md skills instead of install .sh scripts. Why create a complex Software 1.0 bash script … if you can write the installation out in words and say 'just show this to your LLM'

LLM knowledge bases as an example of something that was *impossible* with classical code because it's computation over unstructured data (knowledge) from arbitrary sources and in arbitrary formats

in every new paradigm change, the obvious things are always in the realm of speeding up … but here we have examples of functionality that either suddenly perhaps shouldn't even exist, or was fundamentally not possible before.
— Andrej Karpathy · source
经济模型

Agent 经济的定价逻辑正在重构：Seat 不死，但 Consumption 才是增量所在

Box CEO Aaron Levie 发了一篇干货长文，系统推演了 Agent 时代的软件定价。核心判断：人类用户的 Seat 模式不会消失，但必须捆绑 API 用量，否则就是死路一条。真正棘手的部分是 Agent 的定价——同一个公司可以 1 个 Agent 干所有事，也可以拆成 1000 个细粒度代理，按人头定价完全失效。Levie 的判断是：超过 Seat 配额的用量走 consumption，且 API 会从「按调用次数」向「按 outcome」演进。这对所有面向企业的 SaaS 和 AI 平台都是一个必须面对的架构选择。

查看原文
As agents become the biggest users of software, then all software has to be available in a headless fashion. Agents won't be using your UI, they'll be talking to your APIs.

1) Seats don't go away for *people* … when the customer pays for a seat, it has to come with a set of usage of APIs on behalf of that user that the agent can use.

2) Agents may have 'seats' if they are doing stateful work … the only issue on pricing here is that one customer could decide to do all their work in 1 agent, and another might split it into 1,000 agents.

3) The dominant pricing for headless use … will be a consumption model … there may need to be new types of APIs that represent the work an agent would do in one go—more akin to an outcome—instead of a series of API calls.
— Aaron Levie · source

设计民主化

从 Notion 到 Cursor：系统设计的本质论正在被重新书写

Cursor 设计师 Ryo Lu 发了一条值得深思的推文，谈「opinionated vs general purpose」的对立是假的。他认为「没有意见本身就是一个强烈的意见」——决定哪些概念存在、哪些可见、哪些可组合，就已经表达了世界观。好的系统设计不是在「多功能」和「固定流程」之间选择，而是找到最小的核心概念集（simple at the core, open at the edges），让这些基元足够灵活能够折叠成多种形态而不破坏底层逻辑。这比「以用户为中心」更深一层：用户反馈告诉你摩擦在哪，但很少告诉你底层系统应该是什么。这种思考在 AI 工具的快速迭代期尤其珍贵——当能力边界每天都在变化时，底层概念的设计反而成了产品韧性最关键的因素。

查看原文
no opinion is opinionated. people treat 'opinionated' and 'general purpose' as opposites. they're not. the moment you decide what concepts exist, what's visible, what composes, what extends, what hides – you've already expressed a worldview.

this is also why pure customer-centricity falls short. user feedback tells you where friction is. it rarely tells you what the underlying system should be. if you only follow direct user demand, you converge on local maxima – fragmented features, narrow tools, products shaped around current habits rather than enduring concepts.

simple at the core. open at the edges. that's the ultimate opinion.
— Ryo Lu · source
